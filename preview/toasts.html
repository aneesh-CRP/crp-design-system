Rules of escalation

✓ Toast — the action succeeded, here's confirmation. Auto-dismiss. No blocking.

✓ Banner — state you need to know for this whole session. Never auto-dismiss.

✓ Inline confirm bar — something happened, here's your Undo. Default to this for any delete/remove.

✓ Modal — we need an explicit decision before proceeding. Use sparingly — they block.

✗ Never stack more than 3 toasts — if more are queued, hold them.

✗ Never modal for a success message. Success goes to toast.

✗ Never use toast for errors the user must read — those need a banner or modal.
