Why this pattern works

Every screen in the funnel follows the same contract

01
Progress bar always visible

Step N of M — reduces abandonment by 30% on long funnels (per CRP 2024 landing test).

02
One question per screen

Never multi-column. Never "page of 12 fields." Even on desktop, the funnel stays 420px.

03
Back is a ghost, Continue is primary

Primary action always on the right. Back never styled the same weight as Continue.

04
Radio cards, not radio dots

Entire card is the hit target (44px min). Selected state uses the --blue-bg fill.

Rules

Do

✓ One question per funnel step — no "page of 12 fields"

✓ Always pair a <label> with every input (never placeholder-only)

✓ Mark required fields with the orange asterisk, not "(required)"

✓ Error messages: icon + color + plain-English fix ("Please enter 10 digits")

✓ Consent checkboxes default unchecked — user has to opt in

Don't

✗ Never use placeholder text as the only label — it disappears on focus

✗ Never say "Invalid" or "Error" — tell the user how to fix it

✗ Don't auto-advance the funnel on radio click — confirms reduce errors

✗ No "Submit" on the primary button — use the verb of the next action ("Continue")

✗ Never pre-check marketing opt-ins (GDPR/CAN-SPAM)
